# Plantilla de Pull Request

## Qué cambia

Resume el cambio en pocas frases, separando lo funcional de lo documental cuando aplique.

## Por qué

Explica el problema o necesidad que motiva el PR para que la review no dependa solo del diff.

## Cómo se ha probado

Indica los comandos ejecutados y cualquier comprobación manual relevante, incluso si el cambio es solo documentación.

## Riesgos

Enumera dudas, efectos colaterales posibles o confirma que el riesgo esperado es bajo.
